# Story 2.4 — Agente Publicador (Meta Graph API)

**Status:** Draft

## Descrição

Como AutoPost, preciso que o Agente Publicador publique posts aprovados no Instagram Business e Página do Facebook via Meta Graph API, registre os IDs de publicação e inicie a coleta de métricas 24h depois.

---

## Critérios de Aceitação

- [ ] AC1: Task `publish_post` publica imagem + legenda no Instagram Business via Graph API v19+
- [ ] AC2: Se cliente tem Página do Facebook conectada, publica também no Facebook
- [ ] AC3: `publish_result` salvo com: `instagram_post_id`, `facebook_post_id`, `permalink`, `published_at`
- [ ] AC4: Status transiciona approved → publishing → published (ou failed com error_message)
- [ ] AC5: Task agendada `collect_metrics` criada para rodar 24h após publicação (via Celery Beat)
- [ ] AC6: Token Meta expirado → status = failed + `error_message` com orientação de renovação
- [ ] AC7: Testes com mock da Meta Graph API (httpx mock)

## Dev Notes

- Meta Graph API: `POST /{ig-user-id}/media` → `POST /{ig-user-id}/media_publish`
- Imagem deve estar em URL pública (usar presigned URL do R2 ou URL pública permanente)
- Token Meta expira a cada ~60 dias — adicionar verificação de expiração
- `META_ACCESS_TOKEN` no `.env` é o token de longa duração (Long-Lived Token)
- Métricas: endpoint `/{media-id}/insights?metric=impressions,reach,likes,comments`
