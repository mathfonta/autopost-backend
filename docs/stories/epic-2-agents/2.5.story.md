# Story 2.5 — Agente Onboarding (Claude Sonnet)

**Status:** Draft  
**Epic:** 2 — Agentes de IA  
**Story:** 2.5  
**Estimativa:** 5 pontos  
**Depende de:** 1.2

---

## Descrição

Como novo cliente do AutoPost, quero passar por um fluxo de onboarding guiado por IA que coleta informações sobre minha empresa e cria meu `brand_profile` automaticamente — para que os agentes de copywriting e design usem meu tom de voz e identidade visual desde o primeiro post.

---

## Critérios de Aceitação

- [ ] AC1: Endpoint `POST /onboarding/start` inicia sessão de onboarding para o cliente autenticado
- [ ] AC2: Claude Sonnet conduz conversa estruturada coletando: nome da empresa, segmento, cidade, tom de voz, cores, redes sociais ativas
- [ ] AC3: Ao final, `brand_profile` do cliente é preenchido com os dados coletados e salvo no banco
- [ ] AC4: Fluxo A (tem Instagram): coleta `instagram_business_id` e orienta conexão com Meta
- [ ] AC5: Fluxo B (sem histórico): coleta dados básicos e cria perfil do zero
- [ ] AC6: Onboarding pode ser retomado se interrompido (status salvo em Redis com TTL 24h)
- [ ] AC7: Testes com mock do Anthropic SDK

---

## Tasks

- [ ] Criar `app/agents/onboarding.py` — função `run_onboarding_step(client_id, user_message, session)` 
- [ ] Criar `app/api/onboarding.py` — endpoints: `POST /onboarding/start`, `POST /onboarding/message`, `GET /onboarding/status`
- [ ] Implementar sessão de onboarding em Redis (dict serializado como JSON com TTL 24h)
- [ ] Criar schema de `brand_profile` validado (Pydantic): segment, tone, primary_color, city, instagram_id
- [ ] Atualizar `Client.brand_profile` ao final do onboarding
- [ ] Criar `tests/test_onboarding.py`

---

## Dev Notes

- Claude Sonnet 4.6 (`claude-sonnet-4-6`) para conduzir conversa de onboarding
- Usar `messages` com histórico na sessão Redis (context window management)
- `brand_profile` é JSONB no banco — schema flexível mas validado ao salvar
- Fluxo A vs B determinado por pergunta inicial: "Você já tem Instagram profissional?"
- Prompt do sistema deve incluir lista de segmentos suportados (construção, arquitetura, saúde, etc.)
