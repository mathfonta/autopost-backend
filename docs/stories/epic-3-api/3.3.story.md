# Story 3.3 — Fluxo OAuth Meta (Conectar Instagram/Facebook)

**Status:** Done  
**Epic:** 3 — API de Conteúdo  
**Story:** 3.3  
**Estimativa:** 5 pontos  
**Depende de:** 1.2 (Auth JWT), 1.3 (modelo Client com campos Meta)

---

## Descrição

Como cliente autenticado, preciso conectar minha conta do Instagram Business e Página do Facebook ao AutoPost, para que o Agente Publicador tenha permissão para postar em meu nome.

---

## Critérios de Aceitação

- [ ] AC1: `GET /meta/connect` retorna `{auth_url}` — URL do Meta OAuth com scopes `instagram_basic,instagram_content_publish,pages_manage_posts,pages_read_engagement`
- [ ] AC2: `GET /meta/callback?code=...&state=...` troca o `code` por um Short-Lived Token e em seguida por um Long-Lived Token (válido 60 dias)
- [ ] AC3: Callback salva `meta_access_token`, `instagram_business_id` e `facebook_page_id` no `Client` do banco
- [ ] AC4: `GET /meta/status` retorna `{connected: bool, instagram_username, facebook_page_name, token_expires_at}` — `connected: false` se token não salvo
- [ ] AC5: State param no OAuth contém `client_id` codificado para identificar qual cliente está conectando (CSRF protection)
- [ ] AC6: Se o usuário tem múltiplas Páginas do Facebook, salva a primeira que tiver uma conta IG Business associada
- [ ] AC7: Testes cobrindo geração de auth_url, processamento do callback com mock da Graph API, e consulta de status

## Dev Notes

- Meta OAuth v21.0 — mesmo da Graph API já usada no publisher
- Short-Lived Token dura ~1h; Long-Lived dura 60 dias — salvar o Long-Lived
- Scopes necessários: `instagram_basic`, `instagram_content_publish`, `pages_manage_posts`, `pages_read_engagement`
- Para buscar IG Business ID: `GET /me/accounts` → por cada página, `GET /{page_id}?fields=instagram_business_account` → `instagram_business_account.id`
- `state` para CSRF: usar o próprio JWT de auth do cliente (15 min expiry) — decodificar no callback para saber qual `client_id` está conectando. Não usar o token de longa duração como state.
- `META_REDIRECT_URI` deve estar cadastrado no painel Meta Developers em "Valid OAuth Redirect URIs"
- Long-Lived Token exchange: `GET https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id={app_id}&client_secret={app_secret}&fb_exchange_token={short_lived_token}`
- Token `expires_at`: calcular como `now + 60 days` e salvar em `Client` (adicionar campo `meta_token_expires_at` na migration)
- Falha no callback (code inválido, state inválido) → retornar `400` com mensagem amigável

## Riscos

| Risco | Probabilidade | Mitigação |
|-------|--------------|-----------|
| **Meta App Review obrigatório para produção** — `instagram_content_publish` é um scope avançado que exige aprovação da Meta antes de poder ser usado por usuários que não sejam testadores do app | **ALTA** | **Ação imediata:** cadastrar o app no Meta Developers em modo Development; adicionar clientes como Testers durante o MVP. App Review pode levar 1–4 semanas — iniciar o processo antes de implementar esta story |
| Token expira em 60 dias sem renovação automática | Alta | Documentado no OUT — mitigação v1.1; no MVP, `GET /meta/status` informa `token_expires_at` para o cliente renovar manualmente |
| `META_REDIRECT_URI` não cadastrado no painel Meta Developers | Média | Blocker de configuração — documentar nos Dev Notes; Railway URL deve estar nas "Valid OAuth Redirect URIs" |
| `instagram_business_account` ausente na página (conta pessoal, não Business) | Média | AC6 trata múltiplas páginas; se nenhuma tiver IG Business, retornar mensagem orientando conversão da conta |
